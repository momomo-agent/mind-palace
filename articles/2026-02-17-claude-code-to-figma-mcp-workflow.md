# Claude Code ↔ Figma MCP：从代码渲染态到可编辑图层的工作流

> **作者**: Dylan Field（Figma）
> **来源**: https://x.com/zoink/status/2023760388530307167
> **标签**: AI, Design, Develop

💡 **看点：把 Claude Code 的浏览器渲染状态自动翻译成 Figma 图层，用画布做发散探索，再把设计改动回写代码，避免“只在 IDE 里越写越窄”。**

---

## 发生了什么

Figma 推出一个新的路径：

- 安装 **Figma MCP**
- 在 Claude Code 里输入类似“Send this to Figma”
- **浏览器渲染出来的 UI 状态**会被自动转换成**可编辑的 Figma layers**

从“代码 → 设计稿”不再是截图/手动重画，而是结构化可编辑的图层。

## 他们想强调的观点

Dylan 的核心观点不是“AI 能更快画 UI”，而是：

- AI 让“把任何你能说清楚的东西做出来”变得更容易
- 真正的差异来自 **design / craft / point of view**
- 你在 AI 时代的核心工作，是在近乎无限的可能性空间里找到更好的方案

## 为什么要把画布放回工作流中心

他认为 **design canvas**（Figma 画布）比“在 IDE 里 prompt”更适合：

- 发散探索：并排对比多种方案，看大局
- 直接操作：用拖拽/对齐/视觉校准来打磨细节
- 探索后回写：用 MCP 把设计改动再带回代码仓库

## 关键提醒：避免 tunnel vision

他说现在很容易陷入“创造的惯性”：

- 一旦开始做某个方向，就会自然地一直往下做
- 第一版很容易就变成最终版

所以需要一种机制让你**随时 zoom out**，重新比较路径，确认方向是否正确。
