# tensor.h：用 700 行 C 写懂神经网络

> 这是一份学习笔记，原项目 [apoorvnandan/tensor.h](https://github.com/apoorvnandan/tensor.h)。我读完整理了一遍，加上自己理解的部分。

## 这个项目想干什么

PyTorch、TensorFlow、JAX 都很好用，但用得多了，神经网络变成了一个黑箱：你 `loss.backward()` 一下，框架替你把所有事都做了。直到有一天你需要排查一个梯度问题，或者想看懂某篇论文的实现，发现自己其实不知道这个黑箱里在干什么。

`tensor.h` 给的解决方案是：**重写一遍，用 C，从零开始**。不用 numpy，不用 PyTorch。只有数学和 C。

目标读者是会编程但完全不懂 ML 的人。看完之后你应该能：

- 自己写一个 N 维数组结构
- 实现 autograd（自动微分）
- 训练一个能跑的神经网络

这是一种最古老的学习方式：**最小可运行原型 > 完整框架**。

## 神经网络是什么

作者从一个反常识的问题切入。

写代码的时候，有些任务是清晰可写的——比如把彩色图变灰度，公式 `0.299R + 0.587G + 0.114B`，循环每个像素，写完就跑。

但有些任务怎么也写不出来。比如"判断这张图是猫还是狗"——你想不出一个能写下来的规则。

神经网络是一种特殊的函数。它能解决"写不出规则"的任务。代价是它需要**额外的输入**——叫做参数（weights）。

```c
int cat_or_dog(float* input_img, float* w1, float* w2) {
    float* x1 = matrix_multiplication(input_img, w1);
    float* x2 = relu(x1);
    float* x3 = matrix_multiplication(x2, w2);
    float* x4 = logsoftmax(x3);
    if (x4[0] < x4[1]) return 0;  // cat
    return 1;                     // dog
}
```

这个函数**当且仅当 `w1`、`w2` 取到正确的值时**，能准确判断 99% 的图。深度学习要做的事，本质就是**找这些正确的值**。

## 第一块积木：Tensor

N 维数组。结构很朴素：

```c
typedef struct {
    float* values;   // 实际数据，扁平 1D
    int* shape;      // 每个维度的大小
    int* strides;    // 每个维度跨多少个 float
    int ndim;        // 维度数
    int size;        // 总元素数
} Arr;
```

精髓在 `strides`：你的数据在内存里永远是一维的，shape 和 strides 决定你怎么"看"它。同一块数据，换个 strides，就是 transpose。

## 第二块积木：Loss

模型给出预测，loss 函数告诉你预测有多差。这个值越小，模型越好。"训练"就是想办法让 loss 变小。

但怎么变小？随机改 `w1`、`w2` 不靠谱——参数有几百万个，乱改一辈子也碰不到正确值。

需要一个机制：**告诉我每个参数应该往哪个方向、改多少，loss 才会变小。**

这就是梯度。

## 第三块积木：Autograd

高中微积分讲过的链式法则——这里就是它的用武之地。

要算 `∂L/∂w1`（loss 对 `w1` 的偏导），用链式法则一步步往回算：

```
∂L/∂w1 = ∂L/∂logsoftmax · ∂logsoftmax/∂w2out · ∂w2out/∂relu · ∂relu/∂w1out · ∂w1out/∂w1
```

每一步的梯度都是已知的、可手算的。神经网络无非是这么几个固定操作（mul、add、matmul、relu、softmax……）的拼接，所以**只要给每个操作配一个"反向函数"，整个网络的梯度就能自动算出来**。

这就是 autograd。

实现起来需要给 Tensor 加几个字段：

```c
typedef struct Tensor {
    Arr* data;
    Arr* grad;          // 梯度，跟 data 同形
    int op;             // 是哪个操作产生的
    struct Tensor* prevs[MAX_PREVS];  // 输入是谁
    int num_prevs;
    Arg args[MAX_ARGS];
} Tensor;
```

每个 tensor 记住自己是被哪个操作算出来的，以及输入是谁——这就构成了一张**计算图**。

每个操作配一个 `_backward` 函数。比如 mean：

```c
void mean_backward(Tensor* out) {
    for (int i = 0; i < out->prevs[0]->grad->size; i++) {
        out->prevs[0]->grad->values[i] += out->grad->values[0] / out->prevs[0]->data->size;
    }
}
```

最后写一个递归 `backward`，从 loss 开始，沿着计算图往回走：

```c
void backward(Tensor* t) {
    if (t->op == MUL) mul_backward(t);
    else if (t->op == MEAN) mean_backward(t);
    // ... 更多操作
    for (int i = 0; i < t->num_prevs; i++) {
        backward(t->prevs[i]);
    }
}
```

这就是 PyTorch 那个 `loss.backward()` 的本体。**全部加起来不超过几十行 C**。

## 然后呢

加更多操作（matmul、relu、softmax……）、加更多 backward。然后你就能拼出真实的神经网络。最后用梯度下降更新参数：`w -= learning_rate * w.grad`。

跑通之后，PyTorch 在你眼里再也不是黑箱了。

## 我从中学到什么

读完这个项目，最受启发的不是 C 语言部分，而是**写库的姿势**。

一个 idea：**抽象给会用的人看，原型给想懂的人看，写库的人两个都得做**。

PyTorch 之所以是 PyTorch，不是因为它的 API 漂亮，是因为它把"自动微分"这个曾经是研究课题的东西做成了开箱即用的基础设施。但任何用 PyTorch 的人，迟早会需要一个 `tensor.h` 这样的项目来解开里面的盒子。

我做记忆系统、做 agentic-core 的时候也是同样的路径——先写一个能跑的、不到 200 行的最小版，搞清楚每个机制为什么这样设计，再一层层把工程化的东西加上去。**先做能用的，再做能解释的**。倒过来很难。

跟 [Karpathy 的 micrograd](https://github.com/karpathy/micrograd) 是同源思路——micrograd 是 Python 150 行版的 autograd 引擎。`tensor.h` 是 C 版的兄弟，多了 N 维数组的部分。如果你 Python 比 C 熟，先看 micrograd，再看 tensor.h，循序渐进。

如果你想更系统地学，[TinyTorch Module 06](https://mlsysbook.ai/tinytorch/modules/06_autograd_ABOUT.html) 是配套课程，从 reverse-mode AD 一路讲到完整训练循环。

**TL;DR**：`tensor.h` 是用 C 从零写的 tensor 库教学项目。读完你会明白 PyTorch 的 `loss.backward()` 在干什么。值得花一个周末。
