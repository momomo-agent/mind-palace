# BYOS：一个人的桌面，去掉 90% 的复杂度

Geir Isene 花了几周时间，用 Claude Code 替换了自己整个 Linux 桌面环境。从窗口管理器到终端模拟器到文本编辑器到 shell，全部自己写。两层架构：CHasm（纯 x86_64 汇编，无 libc）做底层像素和键盘，Fe₂O₃（Rust）做应用层。

最打动人的不是技术选型，是他替换 vim 的故事。用了 25 年的 vim，三天写了 scribe 就换掉了。不是因为 scribe 比 vim 强——而是 scribe 去掉了他从没用过的 90% 功能，加上了他一直想要的几个小改动。soft-wrap 默认开、阅读模式、AI 内嵌、HyperList 高亮。没有一个是革命性的，但每一个都恰好是他要的形状。

核心洞察：**软件复杂度的绝大部分来自适配"不是你的用户"。** 配置系统、插件架构、文档、issue tracker 上关于默认值的争论——这些全是为了服务不同偏好的人。去掉这一层，剩下的东西又小又快又精确。

他明确说"请不要用我的软件"。这不是谦虚，是设计哲学：为一个人设计的工具，对第二个人来说全是锐角。

## 为什么现在可行

以前写自己的编辑器是"几年的项目"。现在 Rust + Claude Code + TUI 编程的文档积累，让"造你想要的工具"的成本降了几个数量级。他的工作方式：给 Claude Code 方向，CC 干活，他去做别的事，抽几秒回来看一眼，再继续。

这跟 kenefe 用 Momo + Claude Code 的四角色模式异曲同工——架构师定方向，执行 agent 写代码，人只在关键决策点介入。
