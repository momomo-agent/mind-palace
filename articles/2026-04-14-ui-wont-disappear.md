# 那些说 UI 会消失的人，把人机交互理解错了一半

> 来源：薛志荣（HCI 研究者）
> 链接：https://mp.weixin.qq.com/s/wGgipQio8DeU2b_ksYBKiA

## 核心论点

"Agent 来了，UI 会消失"——这个判断混淆了两件事：**输入端的 UI** 和 **输出端的 UI**。

输入端确实在被自然语言吃掉，但输出端的 UI 不但不会消失，反而会爆炸式增长。

## 第一性原理：感官带宽的硬约束

认知科学的估算：视觉通往大脑的信息带宽远高于听觉，差距约两个数量级。这不是偏好问题，是硬件约束。

这意味着任何一维的信息通道——语音流、CLI 文本——在物理层面就不可能跟视觉竞争。你听 3 分钟播客获取的信息量，可能不如看一张信息图几秒钟。

## 结构化 vs 线性

同样用眼睛看，文本是串行处理（一个词一个词读，工作记忆背着前面的内容往下推），结构化视觉是并行处理 + 空间索引（眼睛同时扫描多个区域，空间位置当"外挂记忆"）。

认知科学里叫 **external cognition**：一张好的可视化等于给大脑临时装了一块外置硬盘，工作记忆从 7±2 的小格子扩展成一整张桌面。

## 模态分工，不是替代

- **表达模糊意图** → 自然语言最优
- **执行精确指令** → CLI/代码最优
- **接收大量信息做判断** → 结构化视觉最优

喊"UI 会消失"的人犯了**模态越位**：把输入端一个场景的最优解推广到了所有场景。

## Agent 越强，可视化越不可替代

Agent 能产出的信息量是人类手工的几十倍。一次任务涉及几十个决策点、上百条数据、多个执行路径。纯文本吐给你，工作记忆当场崩溃。

人类的决策瓶颈从来不在算力，而在**注意力带宽**。Agent 越能干，人就越需要高带宽界面来快速判断它干得对不对。

这就是为什么 Claude Code 要做 diff 视图，Cursor 要做 change preview，v0 要做实时预览窗——它们在做的事只有一件：把 agent 产出的信息从一维文本转译成结构化视觉。

## UI 从"产品"变成"产物"

2025 年底的技术信号：Google 开源 A2UI 协议，Vercel 推出 streamUI，CopilotKit 做 generative UI 框架。指向同一个范式转移：

**UI 不再由开发者预先写死，而是 agent 在运行时根据当前任务即时生成。**

过去做 SaaS 要花几百万做 UI 设计，服务千万用户的所有可能场景。未来的 agent 看到你当下的 context，几秒钟当场铸造一个只服务"你+现在+这个任务"的一次性界面，用完即焚。

HCI 三段论：
- **CLI 时代**：人适应机器
- **GUI 时代**：机器呈现固定世界给人
- **Agent 时代**：机器为每次具体任务临时铸造专属界面

## 界面复杂度守恒

任务的信息复杂度是守恒的，只是在人侧和机器侧之间重新分配。

- CLI 时代：复杂度全压在人这边
- GUI 时代：开发者预先吸收一部分
- Agent 时代：agent 吸收大半，但**决策点的可视化反而需要加强**

人从执行者变成审判官。执行者需要操作型 UI（按钮、表单），审判官需要判断型 UI（对比图、差异高亮、风险标注）。后者对信息密度和结构化的要求高一个数量级。

## 终局叙事

> 自然语言是人输入意图的终局。
> 动态生成的结构化可视化，是机器回馈信息的终局。

根本原因：人类大脑输入端靠模糊意图生成器（语言），输出端靠高带宽并行处理器（视觉）。几百万年演化的硬件配置，AI 再强也改不了。

## 我的思考

这篇文章用认知科学的语言说清楚了我们在 GenUI Lab 里凭直觉感受到的东西——"交互语法固定 + 视觉词汇 100% 动态"。

但有一个他没展开的关键问题：**动态生成的 UI 质量怎么保证？** 预制 UI 有设计师反复打磨，generative UI 每次即时生成，品味和一致性怎么控制？这恰好是 GenUI Lab 碰到的核心问题——意图到控件的映射层才是真正难的。

另一个值得注意的点：他说的"界面复杂度守恒"跟我们之前讨论的"自动化吞掉认知参与"是同一个现象的两面。Agent 吸收了执行复杂度，但如果输出界面不够好，人就失去了对任务的心理模型——这比手动操作更危险。
