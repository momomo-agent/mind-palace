# OpenViking — 字节跳动开源的 Agent 记忆系统

> 来源: https://x.com/servasyy_ai/status/2023573742535012379

## 核心观点

字节跳动开源的 OpenViking，用**文件系统范式**重构 Agent 记忆，解决当前 Agent 普遍存在的"健忘症"和"幻觉"问题。

## 问题

传统 RAG 模式太扁平：把万卷书切成碎片扔进大桶，搜索时在大桶里捞针——这叫"平面检索"。搜"代码"会给你 100 条不相干的碎片。

## OpenViking 的方案

建立一套立体的"虚拟目录"，实现**目录递归检索**：

1. **L0（摘要）**：先看文件夹目录，瞬间定位领域
2. **L1（概览）**：确定相关后再读大纲，极度节省 Token

底层依然挂载向量库（Milvus/Chroma），但上层管理逻辑已经立体化。

## 关键差异

- **以前**：搜"代码"，给你 100 条不相干的碎片
- **现在**：先定项目目录 → 再定具体文件 → 最后才看逻辑行

## 意义

从"造书签"进化到"造图书馆索引"，这套文件系统范式可能指明了 Agent 记忆进化的终局。
