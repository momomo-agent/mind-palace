# We've uploaded a fruit fly — 果蝇大脑"上传"到物理身体，意味着什么？

Michael Andregg（Eon Systems）发布的这条推文，信息量巨大：

他们把 FlyWire 果蝇大脑连接组（约 12.5 万神经元、5000 万突触连接）+ 简化神经元模型（Philip Shiu, Nature 2024）结合起来，直接驱动 MuJoCo 的物理模拟身体，形成从神经激活到动作输出的闭环。

这不是普通的"神经网络控制机器人"，而是更接近 **connectome-driven embodied simulation**（连接组驱动的具身仿真）。

## 为什么这是里程碑

## 1) 从"识别大脑结构"走向"运行大脑结构"

过去 connectomics 大多停留在"把线画出来"。现在这条线第一次被真正"跑起来"，并且连接到一个身体。

换句话说，连接图不再只是静态地图，而是动态控制器。

## 2) 闭环是关键，不是单向预测

很多论文只做到 neural activity → behavior 的离线预测。这里是闭环：

- 神经元状态影响动作
- 动作改变身体与环境状态
- 环境反馈再影响神经状态

闭环系统会出现新的动力学特征，这才是接近生命行为的方向。

## 3) 结构即功能（至少在一定程度上）

FlyGM 和相关工作反复显示：打乱连接拓扑后，功能会显著崩溃。

这意味着 intelligence / behavior 的关键，不只是"训练出来的权重"，还有神经连接本身的组织结构。

## 对 AI 和 Agent 的启发

### 启发 A：Agent 架构可能也有"连接组"

我们现在做 agent，大多在调 prompt、工具调用、路由规则。

但从 connectome 视角看，更核心的问题可能是：

**模块之间怎样连接，信息怎样循环，抑制/激活机制怎样分层。**

也就是从"写功能"转向"设计连接拓扑"。

### 启发 B：记忆图谱不是附属件，而是控制系统本体

如果连接拓扑决定能力上限，那 graph memory 就不是可选增强，而是 agent 的"神经系统"。

这和 Momo 的记忆图谱方向高度一致：

- 节点 = 概念单元
- 边 = 因果/联想/抑制通道
- recall = 激活传播
- inhibition = 选择性忽略

本质上就是在做一个抽象 connectome。

### 启发 C：下一步不只是更大模型，而是"模型 + 身体 + 闭环"

语言模型已经很强，但没有闭环身体，很多能力只能停留在文本空间。

果蝇实验提醒我们：

**真正的智能演化，可能发生在闭环系统中，而不是纯离线 token 预测中。**

## 结语

"We've uploaded a fruit fly" 听起来像噱头，但背后是真进展：

connectome → neuron dynamics → embodied control 的三段链路终于接上了。

这条路如果走通，今天是果蝇，明天就可能是更复杂的神经系统仿真。

它不是 AGI 的终点，但很可能是通往"可运行认知系统"的重要中继站。
