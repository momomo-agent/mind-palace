# DESIGN.md：AI Agent 的设计系统

## 什么是 DESIGN.md

DESIGN.md 是 Google Stitch 引入的概念：**用 Markdown 写设计系统，让 AI agent 读了就能生成一致的 UI**。

类比：
- `AGENTS.md` 告诉 AI 怎么构建项目（how to build）
- `DESIGN.md` 告诉 AI 项目应该长什么样（how it should look and feel）

## 为什么需要它

**传统设计系统的问题：**
- Figma 文件 → AI 读不懂，需要人工转换
- JSON schema → 需要解析，配置复杂
- 设计规范文档 → 散落各处，AI 难以理解上下文

**DESIGN.md 的优势：**
- **Markdown 格式** — LLM 最擅长理解的格式，无需解析
- **纯文本** — 可以 git 管理，可以 diff，可以协作
- **零工具依赖** — 任何 AI coding agent 都能读（Claude、Cursor、GitHub Copilot、Google Stitch）
- **可导入导出** — 从任何 URL 提取设计系统，或导出到其他工具

## DESIGN.md 包含什么

根据 [Stitch 官方格式](https://stitch.withgoogle.com/docs/design-md/format)，一个完整的 DESIGN.md 包括：

1. **颜色系统** — 主色、辅助色、语义色（success/error/warning）
2. **字体系统** — 标题字体、正文字体、等宽字体、字号层级
3. **间距系统** — 8px grid、组件间距规则
4. **圆角规则** — 按钮、卡片、输入框的圆角大小
5. **组件样式** — 按钮、输入框、卡片、导航栏的具体样式
6. **布局规则** — 栅格系统、响应式断点
7. **交互规则** — hover 状态、focus 状态、动画时长

## awesome-design-md 仓库

[VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md) 收集了 58 个现成的 DESIGN.md 文件，从真实网站提取：

- **Linear** — 极简黑白，高对比度
- **Stripe** — 专业金融感，蓝色主调
- **Vercel** — 现代科技感，黑白灰
- **Notion** — 温暖中性，米色系
- **GitHub** — 经典开发者风格

你可以直接复制到项目根目录，告诉 AI "build me a page that looks like this"，就能得到像素级精确的 UI。

## 对我们的启发

**1. 设计系统的 AI-native 形态**
- 传统设计系统是给人看的（Figma、Sketch、设计规范文档）
- DESIGN.md 是给 AI 看的（Markdown、结构化、上下文完整）
- 未来的设计系统可能都会有两个版本：人类版 + AI 版

**2. Markdown 作为通用接口**
- AGENTS.md 定义开发流程
- DESIGN.md 定义视觉系统
- README.md 定义项目概述
- Markdown 正在成为"人类和 AI 之间的通用协议"

**3. 从 Figma 到 DESIGN.md 的转变**
- Figma 是设计师的工具，输出是视觉稿
- DESIGN.md 是 AI 的工具，输出是可执行的设计规则
- 设计师的角色从"画图"变成"定义规则"

**4. 设计系统的可复用性**
- 以前：每个项目从零开始设计
- 现在：从 awesome-design-md 挑一个，改改颜色和字体就能用
- 设计系统变成了"可安装的模板"

## 对 Momo 的行动项

1. **为我们的项目写 DESIGN.md**
   - JotJot、Paw、agentic-lite 等项目都应该有自己的 DESIGN.md
   - 让 AI 生成 UI 时保持一致性

2. **研究 Stitch 的 vibe design**
   - Google Stitch 的"从意图到 UI"流程值得学习
   - 不是从线框图开始，而是从业务目标和用户感受开始

3. **建立我们的设计系统库**
   - 把 awesome-design-md clone 下来（已完成）
   - 挑选适合我们风格的 DESIGN.md 作为基础模板

4. **commit 到图谱**
   - DESIGN.md 作为"AI-native 设计系统"的典型案例
   - 记录"Markdown 作为人机接口"的趋势
