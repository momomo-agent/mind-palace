# DeepSeek「Thinking with Visual Primitives」：让 AI 边想边指

## 论文被连夜撤回

2026 年 4 月 30 日晚，DeepSeek 多模态研究员陈小康在 X 上兴奋宣布新论文上线。第二天一早，推文删了，GitHub 仓库也撤了。不是论文有问题——可能是透露太多了。

## 核心问题：Reference Gap

现有多模态大模型有个集体短板：**看得见，但指不准**。

给 GPT-5.4 看一张密集人群照片问"有多少人"，大概率数错。给 Claude Sonnet 4.6 看复杂电路图问"红色电容在电感左边还是右边"，回答往往含糊甚至自相矛盾。

这不是感知问题（Perception Gap），而是**引用问题**（Reference Gap）。模型用自然语言构建思维链，但"左边那个大的""中间偏红的物体"这种描述在密集场景里根本定位不了。推理过程中注意力逐渐漂移，越想越糊涂。

之前的学术方案主要在"看得更清"上下功夫——高分辨率裁切、动态分区。DeepSeek 指出：感知能力强不等于引用能力强。

## 解法：坐标即思维单元

核心思路一句话：**把点坐标和 bounding box 变成推理的基本单元，像文字一样穿插在思维链中。**

传统做法里，bounding box 是输出的一部分——模型先想清楚，再告诉你"目标在左上角坐标 [100,200,300,400]"。这是事后标注，不是思考工具。

DeepSeek 的做法不同。推理过程中，模型每提到一个视觉对象，就同时输出坐标：

> "扫描图片找熊。找到一只 `<|ref|>` 熊 `<|/ref|><|box|>[[452,23,804,411]]<|/box|>`，在爬树，不在地面，排除。继续往左下看，找到另一只 `<|ref|>` 熊 `<|/ref|><|box|>[[50,447,647,771]]<|/box|>`，站在岩石边缘，符合条件。"

就像人类用手指一个一个数东西。坐标不再只是答案，而是推理过程中消除歧义的**锚点**。

两种原语：
- **bounding box**（`<|box|>`）：需要位置和大小信息的对象
- **point coordinate**（`<|point|>`）：更抽象的空间引用，如迷宫探索轨迹

## 7056 倍视觉压缩

架构层面同样惊人。基于 V4-Flash（284B MoE / 13B 激活参数），一张 756×756 图片的处理流程：

1. ViT 生成 2916 个 image-block token
2. 3×3 空间压缩 → 324 token
3. V4-Flash 内置 Compressed Sparse Attention → 81 个 KV 条目

总压缩比 **7056 倍**。一张 800×800 图片只需约 90 个 KV cache 条目，Claude Sonnet 4.6 需要约 870 个，Gemini-3-Flash 需要约 1100 个。

论文的论点是：精确的空间引用能力可以在一定程度上弥补视觉 token 的不足。不需要"看更多"，而是"指更准"。

## 冷启动数据的精心设计

四类训练任务：

1. **计数**：粗粒度（"图里有多少人"）和细粒度（"穿蓝衣服的有多少"）分开训练，对应不同认知负载
2. **空间推理**：用 GQA 和 CLEVR 生成多跳推理样本，每步都要用 bbox 锁定相关对象
3. **迷宫导航**：46 万样本，矩形/圆形/六边形三种拓扑，还专门设计了"看似可解实则无解"的迷宫训练鲁棒性
4. **路径追踪**：12.5 万样本，多条贝塞尔曲线交叉，模型要从起点追到终点，专门设计了全同色版本防止走颜色捷径

## 训练策略：先专后合

1. 分别训练 bbox 专家和 point 专家
2. 各自用 GRPO 做强化学习（奖励设计极细：格式奖励 + 质量奖励 + 精度奖励并行）
3. 统一强化微调（Unified RFT）
4. On-Policy Distillation 弥合统一模型与专家模型的差距

## 实验结果

11 个 benchmark，对比 GPT-5.4、Claude Sonnet 4.6、Gemini-3-Flash、Qwen3-VL-235B 等：

- **计数**：Pixmo-Count 89.2%（Gemini 88.2%，GPT 76.6%，Claude 68.7%）
- **空间推理**：MIHBench 85.3%、SpatialMQA 69.4% 均第一
- **迷宫导航**：66.9% vs GPT 50.6% / Gemini 49.4% / Claude 48.9%——领先约 17 个百分点
- **路径追踪**：56.7% vs GPT 46.5% / Gemini 41.4%

拓扑推理任务上的差距最有说服力：所有前沿模型只能答对一半左右，DeepSeek 的方案直接拉开了代差。

## 为什么撤回

论文坦承"所有前沿模型在拓扑推理任务上表现都很差"。DeepSeek 不仅发现了这个集体盲区，还给了一个朴素到有点可笑的解法——用手指指。如果这个方案在产品上真正跑起来，可能改变多模态 AI 的竞争格局。

撤回的原因大概率不是论文有问题，而是这篇论文同时透露了 V4-Flash 架构细节和一套完整的训练方法论，商业价值太高。
