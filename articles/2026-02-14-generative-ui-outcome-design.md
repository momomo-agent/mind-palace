# 生成式 UI 与结果导向设计

> 作者：Nielsen Norman Group · 2025

> 标签：AI / Design / Article

**💡 看点：NN/g 对生成式 UI 的权威定义——未来界面将由 AI 实时生成，为每个用户量身定制。设计师的角色将从「画界面」转向「定义约束和用户目标」，这是从「为大众设计」到「为个体设计」的根本转变。**

---

## 什么是生成式 UI

**生成式 UI（GenUI）是由 AI 实时动态生成的用户界面，为用户的需求和上下文量身定制。**

目前的界面必须设计成满足尽可能多的人。任何有经验的设计师都知道这种方式的弊端——你永远无法让所有人完美满意。个性化和自定义能帮上一些忙，但作用有限。

GenUI 提供了从「单一体验设计」到「为每个个体定制体验」的转变潜力。

---

## GenUI vs AI 辅助设计

这是两个不同的概念：

- **生成式 UI**：受益者是终端用户，输出是实时为特定用户生成的动态界面
- **AI 辅助设计**：受益者是设计师和产品团队，输出是 AI 生成的 UI 设计和代码

AI 辅助设计工具（如 UIzard、v0 by Vercel）加速了设计流程，但生成式 UI 对行业的长期影响更为深远。

## 示例：航班浏览

用户 Alex 打开 Delta 航空 App 订机票。她是常旅客，有阅读障碍。

- **无障碍适配**：App 自动使用特殊字体和色彩对比度
- **语音交互**：直接说出需求，系统自动推断出发机场
- **智能预警**：检测到目的地有大型活动，提醒尽早预订
- **个性化排序**：根据她最在意的「价格」和「飞行时间」突出显示
- **偏好过滤**：没有靠窗座位会标注警告，红眼航班自动折叠到底部

**单个案例也许不需要 GenUI，但 Delta 每年 1.9 亿乘客都享受同等个性化体验——这只有 GenUI 才能做到。**

## 转向结果导向设计

**UX 设计传统上聚焦于界面本身。AI 驱动的自动化和生成式 UI 将推动「结果导向设计」的兴起。**

结果导向设计意味着：更关注用户目标和最终结果，同时战略性地自动化交互和界面设计的某些方面。

### 设计师角色的转变

- **从设计界面到设计结果**：微交互的人工设计将变得不那么重要，因为 AI 会让它们变得不必要，或通过 GenUI 动态生成
- **从为大众设计到为个体设计**：设计师将为不同类型的用户定义不同的约束集（guard rails），AI 在生成界面时必须遵守
- **成为参数和约束的设计师**：不再决定每个组件的细节，而是帮助 GenUI 系统理解用户和业务目标
- **研究变得更加关键**：用户研究将比以往更重要，测试确保动态生成的界面有效满足多样化需求

## GenUI 的挑战

- **AI 的问题就是 GenUI 的问题**：幻觉、偏见等问题会直接反映在生成的界面中
- **算力需求巨大**：为数十亿用户实时生成独特界面需要巨大的处理能力
- **隐私风险**：深度个性化需要对用户有深入了解，带来隐私和安全风险
- **一致性丧失**：不断变化的 UI 会导致可用性问题。用户对现代界面的理解根植于设计标准（如 logo 在左上角），每次看到不同界面会增加学习成本

> 设计师需要在完全定制化体验的收益与 UI 一致性和可预测性丧失的代价之间找到平衡。
