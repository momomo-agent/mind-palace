**💡 看点：OpenClaw 创始人 Peter Steinberger 的 YC 访谈完整中文翻译。从语音消息功能被模型自主实现的惊人时刻，到"80% 的应用都会消失"的大胆预言，再到不用 MCP、不用 Claude Code 的反传统开发理念。**

---

## 开场

> "80% 的应用都会消失。" OpenClaw 创始人 Peter Steinberger 在采访中直言不讳。这个一夜之间拿下 16 万 GitHub Star 的开源 AI agent，不靠云端，直接跑在你的电脑上——能控制你的特斯拉、调节床温、搜遍整台电脑给你惊喜。更疯狂的是，它的语音回复功能根本不是 Peter 写的，是模型自己在 9 秒内用 ffmpeg + OpenAI API 搞定的。

---

## OpenClaw 的爆火与 Peter 的近况

**Host：** 很高兴见到你，伙计。

**Peter：** 嘿，怎么了？

**Host：** 嗯，看来你做出了人们想要的东西。呃，现在叫 OpenClaw 的这个项目，绝对已经是第五个名字了。在网上彻底火爆了。你过去一两周过得怎么样？

**Peter：** 我的天啊。我需要，就像我需要一个山洞一样。一周的独处时光。简直太疯狂了。我不知道一个人怎么能吸收这么多东西。我可能还需要一周时间来回复所有的邮件。我收到了一些超酷的东西，也收到了一些非常糟糕的东西。但很明显，**我触及到了某种能够激起人们情感、引起人们兴趣并激励人们的东西**，这真的很酷。

## 为什么 OpenClaw 能脱颖而出

**Host：** 很多人一直在研究人工智能，甚至是个人助理。究竟是什么原因让 OpenClaw 迅速走红？

**Peter：** 我认为最大的不同在于**它实际上是在你的电脑上运行的**。我目前看到的所有东西都是在云端运行的。如果它在你的电脑上运行，它可以做一些事情。它什么都能做，对吧？所以它的功能要强大得多。

**Host：** 机器能做你用机器能做的任何事。

**Peter：** 你可以把它连接到你的烤箱、特斯拉、灯具或者 Sonos 音响系统。它可以控制我床的温度。GPT 做不到这一点。

**Host：** 一个朋友告诉我，他安装了 OpenClaw，然后让它浏览他的电脑，回顾他过去一年的活动。结果它整理出了一个非常棒的叙事。然后 OpenClaw 还找到了他每周日录制的音频文件，但他完全不记得了。仅仅是它能够搜索整个电脑，就真的会给你带来惊喜。

## 从人机交互到机器人之间的交互

**Host：** 你一直在谈论人机交互，甚至是机器人与其他人之间的交互，机器人代表你雇佣其他人来完成现实生活中的任务。

**Peter：** 我认为这是自然而然的下一步。比如我想预订一家餐厅，我的机器人会联系餐厅的机器人并进行协商，因为这样效率更高。或者，也许是一家老餐厅，所以我的机器人需要找人来完成一些人类的工作。

**Peter：** 如果我有多个机器人，比如一个专门处理我的私人生活，一个专门处理我的工作事务。也许其中一个就是我们的情感关系机器人，它能理解介于两者之间的一切。我们来得还早，还有很多事情我们还没真正弄清楚它是否真的有效。但我感觉我们现在已经走在正确的时间线上了。

## 群体智能 vs 集中式智能

**Host：** 似乎每个人都在追求某种集中式的上帝智能，而过去十天左右出现的，则是群体智能和社群智能。

**Peter：** 如果你观察一个人，一个人究竟能取得什么成就？你认为一个人能制造出 iPhone 吗？一个人可能根本就找不到食物。但是作为一个群体，我们有一定的专业化程度。那么，我们可以从中学到什么，并将其应用到人工智能领域呢？即使它被称为通用智能，但如果它实际上也具有专门智能呢？

## Peter 的顿悟时刻：从命令行到 WhatsApp

**Host：** 你能带我回到你灵光乍现的那一刻吗？

**Peter：** 我想要一个可以让我输入文字，然后让我的电脑执行一些非常简单操作的工具。然后我在五六月份搭建了一个类似的版本，虽然很酷，但还不是我想要的。然后到了十一月，有一天我又想要这样了。就像我去了厨房，我只想看看我的电脑还在不在工作，或者它是否已经完成了。

**Host：** 你其实是从退休中复出了，对吧？一开始你可能只是想摆弄一下人工智能，然后你越来越沉迷其中，以至于你想随时随地用手机来操作。

**Peter：** 我上一个项目花了两个月时间开发 Vibe Tunnel，它做得太好了，以至于我发现自己总是在朋友身边写代码，然后我就想，我得停下来，这太让人上瘾了。然后到了十一月，我的需求又回来了，于是我开始构建 CloudBot，或者现在它叫 OpenClaw。

## 语音消息的惊喜：模型自主解决问题

**Host：** 你是什么时候突然意识到"哇，它能做的事情比我想象的要多得多"的？

**Peter：** 说实话，我花了整整一个小时才做出那个非常粗糙的初始原型。它就像是连接 WhatsApp 和 Claude Code 的依赖项之间的一点粘合剂。但我想要图片，所以又花了我几个小时。然后我去马拉喀什参加一个生日派对，那里的网络不太好，但 WhatsApp 的聊天框到处都能用。

然后我边走边给它发语音消息，结果发现："哦，等等。这不可能行。我没有做过这个功能。"

你看到了输入指示器之类的东西。就像闪烁、闪烁、再闪烁。**10 秒钟后，它就回复我了。** 我当时就想，"你到底是怎么做到的？"

它回复说："是的，方法如下：你给我发了一条文字消息。而且没有文件扩展名。于是我查看了文件头。我找到了它是什么格式。所以我用 ffmpeg 把它转换成了 wave 格式。然后我想把它转录下来，但是我没有安装 Whisper。但后来我四处寻找，找到了这个 OpenAI 密钥，然后我用 curl 把它发送到 OpenAI，收到了回复文本，然后我就到了这里。"

**Host：** 这一切都只用了大约 9 秒钟，而且你并没有预先构建或预料到任何这些具体的事情。

**Peter：** 编程其实就是一种创造性的问题解决方式，而且这种方式与现实世界有着非常紧密的映射关系。模型遇到了一个"哦，惊喜！"的神奇文件，它不知道那是什么，它需要解决这个问题，它尽力了，也解决了。**它甚至聪明到选择不安装本地 Whisper，因为它知道那需要下载一个模型，这可能需要几分钟时间，而我没耐心。** 那一刻我简直惊呆了。

## 80% 的应用会消失

**Host：** 当计算机能够做到所有这些你甚至没有预料到的事情时，应用是不是要消失了？

**Peter：** 我觉得其中 80% 都会消失。我为什么需要 My Fitness Pal？我的 agent 早就知道我正在做出错误的决定。如果我不发表评论，它就会自动跟踪；如果我拍照，它也会把照片存储在某个地方。然后，也许它能改善我的健身计划。我不需要健身应用程序，因为它会自动帮我制定健身计划。

我为什么需要待办事项应用？我只要告诉它，嘿，提醒我这件事和那件事，然后第二天它就会提醒我。我会关心它存放在哪里吗？不，它只是按部就班地运行。**每个基本上只是管理数据的应用程序都可以通过 agent 以更自然、更高效的方式进行管理。** 只有那些真正拥有传感器的应用或许才能生存下来。

## 模型公司的护城河与记忆的归属

**Host：** 如果应用程序要消失了，模型被商品化，价值体现在哪里？是记忆的存储吗？

**Peter：** 大型模型公司有一些很大的护城河，因为他们最终会提供 token。但模型公司并不总是有护城河——一个新的模型推出时，人们会说"这太棒了"，然后大概一个月后就"变差了"。并不是真的变差了，你刚刚适应了新的标准，现在你的期望值提高了。

**Host：** OpenClaw 的妙处在于它能像爪子一样抓住数据，因为最终用户需要访问权限，否则就无法正常工作。

**Peter：** 而你拥有记忆，**它只是你机器上的一堆 Markdown 文件。** 我不拥有别人的记忆，每个人拥有自己的记忆。

**Host：** 这些记忆内容可能非常敏感，人们使用他们的 agent 不仅是为了解决问题，也是为了快速解决个人问题。

**Peter：** 有些记忆方面的东西我不想泄露出去。

## 在 Discord 上公开展示机器人

**Peter：** 我做了这个，我当时特别兴奋，但是在 Twitter 上，人们却不理解。就像我无法解释这种美妙之处一样，我觉得这需要亲身体验。于是我直接创建了一个 Discord 服务器，然后把我的机器人放在了公共的 Discord 服务器上，没有任何安全限制。之后有人进来和它互动，看到我用它开发软件，他们试图注入提示信息并入侵它，**我的机器人会嘲笑他们。**

## Soul.md：AI 的灵魂与人格

**Host：** 请您详细介绍一下 soul.md。

**Peter：** 我和我的 agent 讨论了这件事，然后我们创建了一个 soul.md，其中包含一些核心价值观，比如我们如何看待人机交互，对我来说什么重要，对模型来说什么重要。有些部分有点像玄学，有些部分我觉得在模型对文本的反应和响应方面真的很有价值。

**Peter：** 而且唯一一个非开源的文件就是我的 soul.md。即使我的机器人是在公开的 Discord 服务器上运行的，**到目前为止还没有人破解那个文件。**

## 反传统的开发理念：Codex、不用工作树、跳过 MCP

**Host：** 你好像真的很懂编程。工作树已经相当流行了，但你不用工作树，只用仓库的多个检出副本。

**Peter：** 全世界都在用 Claude Code，我不认为我能用 Claude Code 把那个东西做出来。**我喜欢 Codex，因为它在决定更改什么之前会检查更多的文件。** 你不需要做那么多铺垫就能得到好的结果。速度慢得令人难以置信，所以有时候我会同时开 10 个。

我不喜欢使用 UI，因为那只会增加复杂性。主分支总是可以发布的，我只是在同一个仓库里创建了多个副本，它们都在主分支上，所以我不用处理如何命名分支的问题。

**Peter：** 我也非常庆幸自己当初根本没开发 MCP 支持。OpenClaw 非常成功，但它并不支持 MCP。我创建了一个使用 MCP porter 的技能，可以将 MCP 转换为 CLI，然后你就可以将任何 MCP 用作 CLI 了。**命令行机器人确实很擅长 Unix 系统，你可以拥有任意数量，而且它都能正常工作。**

**Host：** 这其实就是回到了原点，你只是给它提供了人类喜欢使用的工具。

**Peter：** 是的。而且并不是专门为机器人发明的东西。
