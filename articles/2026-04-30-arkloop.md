# Arkloop：一个人三个月，做出"有品味"的 AI Agent 客户端

> 读 [@qqqqqf_](https://x.com/qqqqqf_/status/2049776170246471804) 发布 Arkloop 的推文和 [repo](https://github.com/qqqqqf-q/Arkloop) 后的笔记。

清凤（qqqqqf_）一个人做了三个月的 AI agent 客户端，今天发布。看下来有几个点值得讲。

## 不是套壳

作者开篇第一句就挡住了：**不是任何 agent SDK 套壳，也没有任何 base 任何 Claude Code 行为。**

现在 agent 客户端赛道太拥挤，大量是拿 Claude Code / Codex / OpenAI SDK 直接包 UI。清凤这里明确切断这个路径，说"我一个人打磨三个月"。

这种倔强有价值。Agent 核心 / 记忆系统 / hook 机制都开放，你可以接自己的 provider、写自己的插件、甚至改掉任何你不喜欢的设计。

## 设计哲学：减少认知负担

这是我读完最有共鸣的部分。作者举了两个例子：

> "我平常只用一个模型聊天，那为什么每条消息前面都要告诉我用了什么模型？这是噪音。"

> "开发者总是喜欢让 agent 的 tool use 完整的展示到前端，但是背后真正的用户体验逻辑是我需要知道 agent 在工作/在往哪个方向偏，所以我们并不需要如此详细的信息。"

这两条戳中了当下所有 agent 产品的通病。我做 Paw 的时候也一直在想这个问题——工具调用默认全展开还是全折叠？模型名展示还是隐藏？每一个"展示出来"的决定都在跟用户的注意力争抢。

**每多一条信息，用户就多一份认知负担。**

清凤的选择是**默认减少显示**。不是隐藏功能，是重新排序重要性——用户真正需要的是"agent 在做什么、方向对不对"，不是"每一步具体调了哪个函数"。

## 工程上不水

Alpha 阶段，但功能清单比想象中实在：

- **多模型路由** — OpenAI/Anthropic/兼容 API，优先级自动路由 + rate limit 处理
- **沙箱执行** — Firecracker microVM 或 Docker 容器，严格资源限制
- **持久记忆** — 系统约束 / 长期事实 / 会话上下文跨对话保存
- **Prompt Injection 保护** — 语义级扫描检测并阻止注入攻击
- **Channel 集成** — Telegram 媒体处理 + 群组上下文
- **自定义 Personas** — 独立 system prompt / 工具集 / 行为配置，支持 Lua 脚本
- **MCP / ACP** — Model Context Protocol + Agent Communication Protocol
- **Skill 生态** — 从 ClawHub 导入 skill，**兼容 OpenClaw SKILL.md 格式**

Firecracker microVM 做沙箱、Lua 脚本做 persona、Prompt Injection 语义扫描——这些都是没偷懒的信号。一个人三个月做到这个程度，不容易。

## 对我们的意义

**最关键的信号：Arkloop 兼容 OpenClaw SKILL.md 格式，可以从 openclaw/hermes 导入配置。**

这件事值得单拎出来说。OpenClaw 的 skill 格式在设计时其实没有想着做标准——只是 Anthropic 的 Claude Skills 格式的开源实现。但现在有独立产品主动兼容，这意味着：

1. **SKILL.md 格式正在成为事实标准**。有 Claude / OpenClaw / Hermes / 现在 Arkloop 在用
2. **Skill 生态开始跨产品流动**。ClawHub 的 skill 可以装到 Arkloop，Hermes 的配置可以导到 Arkloop
3. **OpenClaw 的价值不只是产品本身，格式和生态更有长期价值**

反过来也给 OpenClaw 提出问题：我们的 skill 格式要不要更正式地标准化？要不要写一个 SKILL.md spec 文档？现在各家都在自己扩展，迟早会出现兼容性问题。

## 我想说的

清凤那段我最喜欢：

> "我欢迎所有形式的贡献。即使你不是开发者，只是一个普通用户——如果你在使用中感到任何不舒服的地方，哪怕只是一点间距、一个颜色、一个很小很小的细节，或者是一个很大的方向，都可以直接开一个 issue。我认真对待每一个体验细节。"

这种对细节的执着，和对非开发者用户的尊重，是独立开发者最宝贵的东西。他还吐槽：

> "我在用 dify 的时候，我常常发现，一个特别小的间距问题，竟然在这么大的仓库里没人修。"

这就是为什么"一个人的产品"依然有空间——大产品的问题不是技术不够好，是**在意的人太少**。一个人的产品能做到每个细节都在意，因为只有一个人在做，那个人就是标准。

## TL;DR

Arkloop 是个开源本地 agent 客户端，一个人打磨三个月。核心是设计哲学：减少认知负担。工程上扎实：Firecracker 沙箱、Lua 脚本 persona、Prompt Injection 保护、MCP/ACP。**兼容 OpenClaw SKILL.md 格式**——这是 agent 生态开始跨产品流动的重要信号。
