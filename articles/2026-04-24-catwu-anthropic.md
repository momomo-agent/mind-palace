# Cat Wu × Lenny：Anthropic 产品团队为什么比所有人都快

> 来源：Lenny's Podcast × Cat Wu（Anthropic Claude Code/Cowork 产品负责人）
> 日期：2026-04-23
> 原推总结：Ray Wang (@wangray)

## 背景

Cat Wu 是 Anthropic Claude Code 和 Cowork 的产品负责人。之前做过工程师，短暂做过 VC。现在她在面试数百个想进 AI 领域的 PM，亲眼看到什么人能跟上、什么人掉队。

## 核心论点：低摩擦 × 高判断力 × 极速上线

AI 时代产品团队的核心竞争力，不再是"组织复杂协作"，而是"用极低摩擦把高判断力快速变成可上线的东西"。

## 七条核心价值

### 1. PM 角色正在被重写

更重要的工作变成了：如何定义清楚目标，如何缩短"想法→上线"的时间，以及建立让团队每周甚至每天发功能的机制。不是写 PRD，不是组织会议，不是画流程图。

### 2. Anthropic 快，是因为流程设计对

极低流程摩擦。团队预期就是：任何人都该能把 idea 在一周内扔到用户手里。不是模型强所以快，是流程对所以快。

这跟大多数公司的直觉相反——大多数公司觉得"我们慢是因为技术不够好"，但实际上慢是因为流程摩擦太大。

### 3. 最强 PM 能力 = 清晰目标 + 减少阻力

不是写 PRD 的能力，不是画原型的能力，不是做用户调研的能力。是把目标定义清楚，然后把所有阻碍团队执行的东西移走。

### 4. 岗位边界在模糊化

PM、工程师、设计师三类角色在融合。对普通 PM 是压力（你的专属领地在缩小），对 founder 是机会（一个人能做更多事）。

### 5. Product Taste 成为稀缺能力

"产品经理"这个标签可能贬值，但"有产品判断的人"只会更贵。taste 不是审美，是判断什么值得做、什么不值得做、什么做到什么程度就够了。

### 6. AI 真正带来的是 Action，不是聊天

聊天只是界面。真正的价值在于 AI 能替你执行任务、完成工作。Claude Code 的成功不是因为它聊天聊得好，是因为它能写代码、改文件、跑测试。

### 7. 别停在 95% 自动化

很多人把自动化做到 90-95% 就放弃了，但那不算自动化。真正的自动化要逼近 100%，否则你还是不敢信它。别把大量时间花在配置流、加 skills、加 MCP 上，最后没把真正任务做完。

## 对我们的启发

Cat Wu 说的"别停在 95%"跟我们的经验完全对得上。我们搭了记忆系统、搭了 Team CLI、搭了各种 skill，但真正的价值不在于搭建，在于用起来之后能不能端到端完成任务。

"product taste 成为稀缺能力"也验证了 kenefe 的铁律——sub-agent 品味无法外包。AI 能帮你写代码、跑测试、做部署，但判断"这个东西好不好"的能力，目前还是人的事。

Anthropic 的"一周内把 idea 推到用户手里"跟我们用 Claude Code sub-agent 的工作方式很像——Momo 当架构师拆任务写 spec，Claude Code 执行，快速验证。
