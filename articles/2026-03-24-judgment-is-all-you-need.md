# 判断力是新的稀缺资源：Will Larson 的四层生产力瓶颈模型

Will Larson（lethain）是《An Elegant Puzzle》的作者，曾在 Uber、Stripe、Calm 做工程管理，现在是 Imprint 的 CTO。这篇文章通过一个具体的工程迁移案例，提出了一个关于 AI 时代生产力瓶颈的清晰模型。

## 同一件事，两个时代

Will 对比了两次技术迁移：

**2014 年 Uber**：把数百个服务迁移到新的自建服务平台。~3 人团队，6 个月，"我不记得在这六个月里有过任何停歇"。那时 Kubernetes 还不存在，他们从零构建调度算法、服务发现、部署流程。绝大部分时间花在**实现决策**——决定了方案之后，一行一行把它写出来。

**2026 年 Imprint**：把所有服务和数据库迁移到持续部署。同样 ~3 人团队，3 个月。基础设施已经有了（Kubernetes、ArgoCD），问题变成如何组合这些平台。绝大部分时间花在**设计方案、审查 coding agent 的 PR、在设计和现实不匹配时修正方案**。

> 2014 年我们把绝大部分时间花在"实现决策"上；2026 年我们几乎全部时间都花在"设计方案"上。

同样规模的团队，执行时间砍半。但关键变化不是速度——是**时间花在了完全不同的地方**。迁移的核心挑战（做对架构决策、处理边界情况、协调依赖关系）没有变，但 2026 年团队只需要解决这些挑战，不再需要处理"实现那些决策的繁琐细节"。

## 四层生产力瓶颈

基于这次迁移经验，Will 提出了一个四层模型：

### 第一层：时间 — 已解决

Coding agents 已经基本解决了时间问题。"我们有效地拥有无限的时间，以非常可负担的价格，来完成我们的工作。" 这是最底层的约束，也是最先被解除的。

### 第二层：注意力 — 正在解决

Will 说他同时管理超过 5 个并发项目时会丢失追踪。但他认为这是工具组合的问题（"a skill issue in how I am composing the tools"），不是固有限制。注意力瓶颈不会降到零——团队协作本身需要注意力协调——但会大幅降低。

### 第三层：判断力 — 当前最大瓶颈

> 有了无限时间，注意力约束逐渐减少，我可以做任何事。但我怎么做得可维护、安全、可靠？怎么做到一个关键工程师离职后系统还能跑？

这是现在最大的瓶颈。执行免费了，但**做对决策**仍然昂贵。

Will 提出的解决方案是 **datapacks / skill package manager**：把专家判断力打包成可分发的 skills，注入到 coding agent 的工作上下文中。他预测行业会发展出高质量 skills 生态——"你可以很容易想象 O'Reilly 或另一家技术出版商开发一个 blessed skills 的包管理器"。

这个概念已经在多个层面发生了：
- 社区层：ClaWHub 这样的 skill 分发平台
- 公司内部：企业自建的内部 skills（安全工程、产品工程等）
- 开发工具层：AGENTS.md、CLAUDE.md 这类项目级指令

### 第四层：创造力 — 未来问题

Will 认为这个问题"足够远，我不太担心"，而且它本质上是创业问题，现有的解法仍然适用。

## 这条链路的完整拼图

这篇文章跟最近几条收藏形成了一条清晰的思路链：

**Cat Wu（Claude Code 产品负责人）**说 PM 的核心从"定义确定性"变成了"加速发现"——因为你定义的约束可能半个月后就不存在了。

**kenefe 的洞察**："研发未来 = 做决策"——执行向岗位在消失，AI 做所有该做的事，人做所有想做的事。

**Carlini C Compiler 经验**：16 个 agent 并行，"test quality > everything"，不需要编排 agent（每个 agent 自己决定"下一个最明显的问题"）。

**Will Larson 的四层模型**把这些拼在了一起：时间免费了 → 注意力在解决 → 判断力是当前瓶颈 → 创造力是终极问题。

而判断力的规模化方案——skill package manager——跟 ClaWHub、GStack、网易云的 Agent Skills 都是同一个方向：**把专家经验打包成 agent 能消费的格式，让判断力也变成可分发的资源。**

这可能是软件工程接下来 2-3 年最大的结构性变化：不是 AI 替代程序员，而是**判断力成为唯一真正稀缺的东西**，围绕判断力的生产、分发、消费会重塑整个行业。
